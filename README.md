## Дипломный проект. Задание 1: Юнит-тесты

### Автотесты для проверки программы, которая помогает заказать бургер в Stellar Burgers

### Реализованные сценарии

Созданы юнит-тесты, покрывающие класс `Burger`

Процент покрытия 100% (отчет: `htmlcov/index.html`)

### Структура проекта

- `praktikum` - пакет, содержащий код программы
- `tests` - пакет, содержащий тесты, разделенные по классам.

### Запуск автотестов

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов и создание HTML-отчета о покрытии**

>  `$ pytest --cov=praktikum --cov-report=html`

**Tests**
>1. test_set_buns()</h>

Корректность установки булочки в бургер.
Что булочка действительно сохраняется в атрибуте bun.

>2. test_add_ingredient()</h>

Корректность добавления ингредиента.
Что ингредиент появляется в списке ingredients.

>3. test_remove_ingredient()</h>

Корректность удаления ингредиента по индексу.
Что список ингредиентов обновляется правильно.

>4. test_move_ingredient()</h>

Корректность перемещения ингредиента.
Что порядок ингредиентов меняется как ожидается.

>5. test_get_price() (параметризованный)</h>

Правильность расчета общей стоимости в разных сценариях:
Только булочки (цена × 2).
Булочки + несколько ингредиентов.
Дорогая булочка + много ингредиентов.

Параметры теста:

python
@pytest.mark.parametrize("bun_price,ingredients_data,expected_price", [
    (100, [], 200),           # Только булочки: 100×2 = 200
    (100, [(200,), (300,)], 700),  # 100×2 + 200 + 300 = 700
    (200, [(100,), (200,), (300,)], 900)  # 200×2 + 100 + 200 + 300 = 900
])

>6. test_get_receipt()</h>

Полное соответствие формата чека:
Булочки сверху и снизу.
Ингредиенты между ними.
Итоговая цена.
Корректность переносов строк и форматирования.
Ожидаемый вывод.

>7. test_get_receipt_empty_ingredients()</h>

Формат чека без ингредиентов.
Что цена считается правильно (только булочки × 2).
Ожидаемый вывод.
