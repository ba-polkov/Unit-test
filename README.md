# Тесты для проекта Stellar Burgers

Этот репозиторий содержит автоматизированные тесты для компонентов системы Stellar Burgers, включая тесты для булочек (Bun), ингредиентов (Ingredient), бургеров (Burger) и базы данных (Database).

## Структура тестов

### 1. `test_bun.py`
Тестирует класс `Bun` и его методы:
- Создание булочки с различными параметрами
- Проверка типов возвращаемых значений
- Методы `get_name()` и `get_price()`

**Ключевые тест-кейсы:**
- Проверка корректности создания булочки
- Проверка типов данных (имя - str, цена - float)

### 2. `test_ingredient.py`
Тестирует класс `Ingredient`:
- Инициализация с разными типами ингредиентов (соус, начинка)
- Методы `get_type()`, `get_name()` и `get_price()`

**Ключевые тест-кейсы:**
- Проверка создания ингредиентов всех типов
- Проверка возвращаемых значений методов

### 3. `test_burger.py`
Тестирует класс `Burger`:
- Инициализация бургера
- Добавление/удаление ингредиентов
- Перемещение ингредиентов
- Расчет цены
- Формирование чека

**Ключевые тест-кейсы:**
- Тестирование добавления и удаления ингредиентов
- Проверка обработки неверных индексов
- Тестирование перемещения ингредиентов
- Проверка расчета общей стоимости
- Проверка формата чека

### 4. `test_database.py`
Тестирует класс `Database`:
- Доступные булочки и ингредиенты
- Проверка типов возвращаемых объектов
- Проверка кэширования данных

**Ключевые тест-кейсы:**
- Проверка корректности данных булочек и ингредиентов
- Проверка, что возвращаются объекты нужных классов
- Проверка, что возвращается один и тот же экземпляр списка

## Зависимости

Для работы тестов требуется:
- Python 3.7+
- pytest
- pytest-mock (для тестов с моками)

Установка зависимостей:
```bash
pip install pytest pytest-mock
```

## Запуск тестов

1. Запуск всех тестов:
```bash
python -m pytest -v
```

2. Запуск тестов с генерацией отчета pytest -cov:
```bash
pytest --cov=praktikum --cov-report=term-missing --cov-report=html tests

```

3. Запуск конкретного тестового файла:
```bash
pytest tests/test_burger.py -v
```

## Фикстуры и моки

Проект использует:
- Фикстуры pytest для подготовки тестовых данных
- Моки для изолированного тестирования компонентов
- Параметризованные тесты для проверки различных сценариев

## Данные для тестирования

Все тестовые данные вынесены в модуль `data.py`, который содержит:
- Тестовые данные для булочек (`DataBun`)
- Тестовые данные для ингредиентов (`DataIngredients`)
- Тестовые данные для бургеров (`DataBurger`)
- Тестовые данные для базы данных (`DataDatabase`)

## Особенности реализации

1. **Параметризованные тесты** - позволяют проверять множество сценариев с минимальным дублированием кода
2. **Моки** - используются для изолированного тестирования компонентов
3. **Строгая проверка типов** - все тесты включают проверки типов возвращаемых значений
4. **Полное покрытие** - тесты охватывают все основные сценарии работы каждого компонента
