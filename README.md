## Дипломный проект. Задание 1: Юнит-тесты

### Автотесты для проверки программы, которая помогает заказать бургер в Stellar Burgers

### Реализованные сценарии

Созданы юнит-тесты, покрывающие классы `Bun`, `Burger`, `Ingredient`, `Database`
Процент покрытия 100% (отчет: `htmlcov/index.html`)
В тестах задействованы, моки, фикстуры и параметризация в соответствии с заданием
Тесты независимы
В requirements.txt только используемые зависимости 
Неиспользуемых импортов нет
Закомментированных блоков кода нет

### Структура проекта

- `source_code` - пакет, содержащий код программы
- `tests` - пакет, содержащий тесты, разделенные по классам. Например, `bun_test.py`, `burger_test.py`, `ingredient_test.py`, `database.py`
- `data` - пакет, содержащий тестовые данные 
- `conftest` - моки для тестов и прочие методы вынесены в фикстуры
- `pytest.ini` - файл для принудительного установления working directory, т.к. в некоторых созданных файлах сбивается 
- `__init__.py` - файлы во всех папках и корневой директории, чтобы объединить модули 

### Запуск автотестов

> pytest -v (в терминале)

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов и создание HTML-отчета о покрытии**

>  $ pytest --cov=source_code.bun --cov=source_code.ingredient --cov=source_code.burger --cov=source_code.database --cov-report=html (т.к. в папке есть сторонние файлы, которые не нужно покрывать тестами в соответствии с заданием)
